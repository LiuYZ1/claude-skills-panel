## 1. 项目概述

### 1.1 项目目标
在 VS Code 侧边栏中创建一个专用的功能面板，集中展示所有 Claude Code 技能（Slash Commands），支持搜索、翻译、标签、AI 自动分类、一键复制调用，提升开发效率。

### 1.2 适用平台
- VS Code 1.120.0 及以上
- Windows / macOS / Linux

### 1.3 开发环境
| 环境 | 版本 |
|------|------|
| VS Code | 1.124.2 |
| Node.js | 22.13.1 |
| TypeScript | 5.9.3 |
| @types/vscode | 1.120.0 |
| @types/node | ^25.9.3 |
| @vscode/vsce | 3.9.2 |

## 4. 模块详解

### 4.1 `extension.ts` — 扩展入口

注册 WebviewViewProvider 和全局命令：
- `claude-skills-panel.copySkill`
- `claude-skills-panel.sendToClaude`
- `claude-skills-panel.refreshSkills`
- `claude-skills-panel.importSkill`

### 4.2 `skillsData.ts` — 数据中心

核心功能模块：

| 模块 | 功能 |
|------|------|
| SkillItem 接口 | command, label, description, group, source, labelCn, descCn, alias, tags |
| BUILTIN_SKILLS | 13 个 Claude Code 原生技能 |
| scanLocalSkills() | 扫描 ~/.claude/skills/ 目录，解析 SKILL.md |
| parseSkillMd() | 支持单行/多行 YAML frontmatter 解析 |
| translateSkill() | 调用子进程 translate-cli.js 翻译描述 |
| classifySkills() | 调用子进程 classify-cli.js AI分类（每批15个） |
| setAlias/getAliases | 别名管理（存储到 JSON） |
| setSkillTag/getSkillTags | 标签管理（存储到 JSON） |
| importSkillFromGit | git clone 导入技能 |
| removeLocalSkill | 删除本地技能目录 |
| groupSkills() | 按 tag/ai 模式分组 |

### 4.3 `htmlTemplate.ts` — Webview UI

生成完整的 HTML/CSS/JS 页面，包含：

**布局组件：**
- 顶部工具栏（刷新 + 导入按钮）
- 分类模式切换（🏷️ 标签 / 🤖 AI分类）
- 过滤标签栏
- 搜索框
- 可折叠分组列表
- 技能卡片网格

**卡片交互：**
- 点击卡片 → 复制命令
- 🏷️ → 设置标签（弹出 VS Code 输入框）
- 📄 → 打开 SKILL.md
- ✎ → 设置别名
- ✕ → 移除技能（本地）
- 🌐 → 翻译描述

**状态持久化：**
- 使用 `vscode.getState()/setState()` 保存展开状态
- 重绘后自动恢复

### 4.4 `skillsProvider.ts` — 视图提供器

处理所有 Webview 消息：

| 消息类型 | 操作 |
|---------|------|
| skillClicked | 复制命令到剪贴板 |
| refreshSkills | 重新扫描本地技能 |
| importSkill | 从 Git 导入技能 |
| setAlias | 设置别名 |
| setSkillTag | 设置标签 |
| translateSkill | 翻译描述 |
| classifySkills | AI 分类未归类技能 |
| viewSkillFile | 打开 SKILL.md |
| removeSkill | 移除本地技能 |
| changeGroupMode | 切换分组模式 |

### 4.5 子进程脚本

#### `translate-cli.ts`
- 独立 Node.js 子进程
- 读取 settings.json 获取 API 配置
- 调用 DeepSeek API 翻译描述
- 输出 JSON 结果到 stdout
- 支持 `thinking` + `text` 双 block 响应解析

#### `classify-cli.ts`
- 从 stdin 读取技能数据（避免命令行长度限制）
- 调用 DeepSeek API 分类
- 15 个预定义类别
- 输出 `{command: category}` JSON 到 stdout

## 6. 安装与使用

### 6.1 安装方式

#### 方式一：命令行安装
```bash
code --install-extension claude-skills-panel-1.0.0.vsix
```

#### 方式二：VS Code 界面安装
扩展管理 → ··· → 从 VSIX 安装

#### 方式三：双击安装
Windows 直接双击 `.vsix` 文件

### 6.2 使用方法

#### 基础操作
| 操作 | 效果 |
|------|------|
| 点击活动栏 ⚡ 图标 | 打开技能面板 |
| 点击技能卡片 | 复制 `/命令` 到剪贴板 |
| 搜索框输入 | 按名称/命令/描述模糊搜索 |

#### 分类模式
| 模式 | 说明 |
|------|------|
| 🏷️ 标签 | 按用户设置的自定义标签分组 |
| 🤖 AI分类 | 按 AI 自动分析的用途分类（15 个类别） |

#### 卡片按钮
| 按钮 | 功能 |
|------|------|
| 🏷️ | 输入标签名（已存在则移除） |
| 📄 | 打开 SKILL.md 到编辑器 |
| ✎ | 设置中文别名 |
| ✕ | 移除本地技能（仅本地安装） |
| 🌐 | 翻译描述（在描述文字旁） |

#### AI 分类
1. 点击顶部 🧠 按钮
2. 每次分析 15 个未分类技能
3. 多点几次直到全部分类完成
4. 自动切换到 🤖 AI分类 模式

#### 导入技能
1. 点击顶部 📥 按钮
2. 输入 GitHub 仓库 URL
3. 自动 git clone 到 `~/.claude/skills/`

### 6.3 系统要求
| 项目 | 要求 |
|------|------|
| VS Code | 1.120.0 及以上 |
| 操作系统 | Windows / macOS / Linux |
| API Key | 需配置 Claude/DeepSeek API（用于翻译和分类） |

### 6.4 卸载方法
```bash
code --uninstall-extension claude-user.claude-skills-panel
```

## 7. 安全设计

### CSP 策略
```
default-src 'none'                          // 禁止外部资源
style-src ${cspSource} 'unsafe-inline'      // 内联样式
script-src ${cspSource} 'unsafe-inline'     // 内联脚本
```

### 权限最小化
- 仅 clipboard.write（自动授权）
- 调用 claude-vscode.* 公开命令
- 无文件系统访问（除读取 settings.json 和 skills 目录）
- API 调用通过子进程，不阻塞扩展主线程

### Webview 隔离
- `localResourceRoots: []` — 无本地资源加载
- VS Code webview 沙箱隔离 DOM
- 仅通过 postMessage 通信
